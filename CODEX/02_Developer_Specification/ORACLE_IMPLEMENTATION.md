# CODEX-008 - Implementacion del Oraculo

**Fuente:** CODEX-008 El Oraculo v1.0.0  
**Estado:** contrato estricto aprobado  
**Ultima actualizacion:** 2026-07-21

## Limites de autoridad

El Oraculo recibe una entrada, fecha, hora, idioma y contexto relevante ya seleccionado por el Motor. No recibe credenciales ni consulta la base de datos. Su salida util es JSON validado; la Cronica es narrativa y nunca altera los hechos originales.

El contrato no admite puntos de estadistica, XP, monedas, dano, transiciones de contratos o modificaciones de Disciplina. Puede proponer actividades clasificadas, entidades, emociones y evidencia. El Motor valida y decide cualquier cambio persistente.

## Contexto incompleto

Los arreglos de propuestas pueden quedar vacios cuando no exista evidencia suficiente. Es preferible una propuesta incompleta a informacion inventada. Una respuesta fuera del contrato se registra como rechazada y el Diario original permanece intacto.
